# プロンプトの基礎

## はじめに

プロンプトとは、AIモデルに対して与える指示や質問のことです。適切なプロンプトを書くことで、AIからより正確で有用な回答を引き出すことができます。この記事では、効果的なプロンプトを書くための基礎知識を解説します。

---

## 1. プロンプトとは何か

プロンプト（Prompt）は、LLM（大規模言語モデル）への入力テキストです。AIは受け取ったプロンプトを解釈し、それに基づいて回答を生成します。

プロンプトには主に以下の要素が含まれます：

- **指示（Instruction）**: AIに何をしてほしいかを伝える
- **コンテキスト（Context）**: 背景情報や前提条件
- **入力データ（Input Data）**: 処理してほしい具体的なデータ
- **出力形式（Output Format）**: 期待する回答の形式

---

## 2. 基本的なプロンプトの書き方

### 2.1 明確で具体的に書く

曖昧な指示はAIの出力を不安定にします。

**悪い例：**
```
何か面白いことを教えて
```

**良い例：**
```
日本の歴史における江戸時代の主な特徴を3つ、箇条書きで教えてください。
```

### 2.2 役割を与える（ロールプロンプティング）

AIに特定の役割を演じさせることで、専門的な回答を引き出せます。

```
あなたは経験豊富なPythonエンジニアです。
以下のコードのバグを見つけて修正してください。

[コード]
```

### 2.3 出力形式を指定する

期待する形式を明示することで、回答の一貫性が向上します。

```
以下のトピックについて説明してください。

形式：
- 概要（2〜3文）
- メリット（箇条書き3点）
- デメリット（箇条書き2点）

トピック: リモートワーク
```

## 3. プロンプトの主要テクニック

### 3.1 Few-shot プロンプティング

例を示すことで、AIが期待するパターンを学習します。

```
以下の例を参考に、文章をポジティブに書き換えてください。

例1:
入力: 今日は疲れた
出力: 今日もよく頑張った

例2:
入力: 失敗してしまった
出力: 次への貴重な学びを得た

タスク:
入力: 時間がなかった
出力:
```

### 3.2 Chain-of-Thought（思考の連鎖）

複雑な問題は「ステップバイステップで考えて」と指示することで精度が上がります。

```
以下の問題をステップバイステップで解いてください。

問題: 1から100までの整数の合計はいくつですか？
```

### 3.3 制約を設ける

回答の範囲や条件を絞ることで、的を外れた回答を防げます。

```
以下の制約に従って回答してください：
- 200文字以内で答えること
- 専門用語は使わないこと
- 小学生にも分かる言葉を使うこと

質問: 光合成とは何ですか？
```

## 4. システムプロンプトとユーザープロンプト

モダンなLLM APIでは、プロンプトは役割ごとに分かれています。

| 種類 | 説明 | 使用例 |
|------|------|--------|
| **システムプロンプト** | AIの振る舞いや役割を定義 | 「あなたはカスタマーサポート担当者です」 |
| **ユーザープロンプト** | ユーザーからの具体的な質問・指示 | 「返品ポリシーを教えてください」 |
| **アシスタントプロンプト** | AIの過去の回答（会話履歴） | 前の回答の続きを促す際に使用 |

## 5. よくある失敗パターンと改善策

### 5.1 指示が抽象的すぎる

| 問題 | 改善 |
|------|------|
| 「良い文章を書いて」 | 「ビジネスメール形式で、件名・本文・署名を含む200字程度の文章を書いて」 |
| 「コードを直して」 | 「このPythonコードのTypeErrorを修正して。エラーメッセージ: ...」 |

### 5.2 コンテキストが不足している

AIは会話の外部の情報を知りません。必要な背景情報は明示的に提供してください。

**改善前：**
```
この件についてどう思いますか？
```

**改善後：**
```
先日発表されたOpenAIのo3モデルについて、
従来のGPT-4との主な違いを教えてください。
```

### 5.3 複数の指示を一度に詰め込む

複雑なタスクは小さなステップに分解しましょう。

## 6. 実践的なプロンプトテンプレート

### 文章要約

```
以下の文章を{文字数}字以内で要約してください。
重要なポイントを漏らさず、簡潔にまとめてください。

文章:
{要約したい文章}
```

### コードレビュー

```
以下のコードをレビューしてください。

観点：
1. バグや潜在的なエラー
2. パフォーマンスの改善点
3. 可読性・保守性

言語: {プログラミング言語}
コード:
{コード}
```

### アイデア出し

```
{テーマ}について、ユニークなアイデアを{個数}個提案してください。

条件：
- {条件1}
- {条件2}

各アイデアには簡単な説明（1〜2文）をつけてください。
```

## 7. プロンプトエンジニアリングのベストプラクティス

1. **反復改善**: 最初から完璧なプロンプトを目指さず、結果を見ながら改善する
2. **バージョン管理**: 効果的なプロンプトはドキュメントとして保存する
3. **テスト**: 同じプロンプトを複数回実行して安定性を確認する
4. **シンプルさを保つ**: 必要以上に複雑にしない
5. **ネガティブ指示の活用**: 「〜しないこと」という指示も有効

---

## まとめ

プロンプトの質は、AIから得られる回答の質に直結します。

- **明確・具体的**に書く
- **役割・形式・制約**を指定する
- **例示**で期待するパターンを伝える
- **ステップバイステップ**で複雑な問題を解く

これらの基礎を押さえることで、AIをより効果的なツールとして活用できるようになります。継続的に試行錯誤しながら、自分のユースケースに最適なプロンプトを見つけていきましょう。
