# 執筆ルール

このリポジトリで記事を作成・編集する際は、以下のルールに従って文章を書くこと。

---

## 1. 基本的な文体

- **語りかけるです/ます調**を使う。読者に直接話しかけているような文体にする
- `〜んです`・`〜ですよ`・`〜ですよね`・`〜ましょう` を自然に混ぜる
- `！` を適度に使って、親しみやすさとテンポを出す
- `…`（三点リーダー）を列挙の末尾や例示の後に使い、余韻やリズムを出す
- 箇条書き内では `→` 矢印を使って「原因 → 結果」「入力 → 出力」などを簡潔に示す

---

## 2. 記事の構成

### 冒頭（はじめに）

- 読者が共感できる「あるある」な問いかけや失敗例を **引用符（「」）** で示す
- 直後に `そんな経験、ありませんか？` のような問いかけで共感を引き出す
- 「この記事で何がわかるか」を1〜2文で予告する
- `一緒に見ていきましょう！` など、読者を誘うひと言で締める

**例:**
```
「なんかうまくいかない…」「どうすればいいんだろう？」

そんな経験、ありませんか？

実は、〇〇を知っておくだけでぐっと変わります。基本さえ押さえればすぐに使えるものばかりです。一緒に見ていきましょう！
```

### セクション1：概念の定義

- タイトルは `そもそも「〇〇」って何？` の形式にする
- 専門用語には英語表記を併記する。例: `構図（英語では Composition）`
- 難しくなりすぎないよう、日常の具体例で補足する
- 要点を箇条書きでまとめ、最後に `これだけ！シンプルですよね。` のような一言を添える

### セクション2以降：テクニック・方法の解説

- タイトルに数字と感嘆符を使う。例: `これだけで変わる！基本の構図5選`
- 各テクニックは以下の流れで書く:
  1. 1〜2文で定義
  2. コードブロックやASCIIアートで視覚的に示す（必要な場合）
  3. `> **実践のコツ:**` または `> **ちょっとしたコツ:**` のブロッククォートで実用的なTipsを追加
  4. 締めに読者への行動促進フレーズ（`ぜひ試してみてください` など）を入れる

### 失敗パターンのセクション

- タイトルは `あるある失敗パターン、こう直す！` のようにする
- 各パターンは `「〜してしまう」問題` という形式で見出しを立てる
- ブロッククォート形式で原因と解決策を示す:

```
> **原因:** 〜
> **解決策:** 〜
```

### まとめ

- `構図の基礎をおさらいしておきましょう！` のように、`おさらい` `振り返り` のニュアンスで始める
- **太字** のキーワード + `→` + 一言説明の箇条書きで要点を列挙する
- 最後は「まず〇〇だけ意識するところから始めてみてください。きっと〜ですよ！」のように、ハードルを下げて背中を押すひと言で締める

### 末尾

必ず以下の形式で作成日を記載する:

```
*作成日: YYYY年M月D日*
```

---

## 3. Markdownの使い方

| 要素 | 使い方 |
|------|--------|
| `**太字**` | キーワード・重要な概念・強調したい言葉 |
| `> **コツ:** 〜` | Tips・実践アドバイスをブロッククォートで目立たせる |
| `---` | セクション間の区切り線（各セクションの末尾に入れる） |
| コードブロック | 例文・テンプレート・ASCII図の表示 |
| テーブル | 比較情報・一覧表（列ヘッダーは親しみやすく。例: `雰囲気`、`こんなときに`） |
| `→` | 箇条書き内の原因→結果、入力→出力などの関係 |

---

## 4. セクション見出しのパターン

- 概念定義: `そもそも「〇〇」って何？`
- テクニック一覧: `これだけで変わる！〇〇 N選`
- 応用・発展: `「〇〇」を使いこなすテクニック`
- 失敗と改善: `あるある失敗パターン、こう直す！`
- 練習・上達法: `〇〇を上達させるコツ`
- 目的別ガイド: `伝えたい気持ちに合わせて〇〇を選ぼう`
- まとめ: `まとめ`（シンプルに）

---

## 5. 避けるべき表現・スタイル

### 文体
- ❌ 「本記事では〜を解説します」→ ✅ 冒頭の共感フレーズから入る
- ❌ 「〜となります」「〜においては」→ ✅ 「〜です」「〜では」
- ❌ 「〜することが可能です」→ ✅ 「〜できます」
- ❌ 「〜することができます」→ ✅ 「〜できます」
- ❌ 「〜と言えるでしょう」→ ✅ 「〜ですよ」「〜と思います」
- ❌ 「非常に」「極めて」などの大げさな強調→ ✅ 「とても」「かなり」「ぐっと」

### 構成
- ❌ 定義だけ書いて終わる（具体例がない）
- ❌ 箇条書きが連続しすぎて文章がない
- ❌ セクションタイトルが無味乾燥（「〇〇について」「〇〇の方法」など）
- ❌ まとめが単なる内容の繰り返しで締めの言葉がない
- ❌ 読者への行動促進フレーズがない

---

## 6. トーンのバランス

- **親しみやすい ≠ 軽すぎる**: 専門用語はしっかり使い、英語表記も併記する
- **情報量は落とさない**: 口語調でも内容の正確さ・網羅性は維持する
- **読者を信頼する**: 過度な説明はせず、「ぜひ試してみてください」など行動を促す
- **ポジティブなトーン**: 失敗パターンを紹介するときも「こう直せばOK！」で終わらせる
