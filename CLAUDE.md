# 執筆ルール（文体・構成ガイド）

このドキュメントは、記事/ 配下の実際の記事を分析して抽出した、執筆スタイルのルールです。
AIが記事を生成・補完する際は、このルールに従ってください。

---

## 1. 書き出し・挨拶

- **必ず「どうも」で始める**（"こんにちは"や"はじめに"などは使わない）
- 続けて近況・きっかけ・今日書く理由を1〜2文で入れる
- 例:
  - `どうも、僕です。先週のスナップ記事です。`
  - `どうも。土日はペット撮影イベントのカメラマンをしてきた僕です。`
  - `どうも僕です。本日の記事は〜のレビューです。`

---

## 2. 一人称・視点

- 一人称は常に **「僕」**
- 読者への語りかけは **「〜ですよね」「〜じゃないですか」** で巻き込む
- 体験・気づきは **自分の実体験をベース** にして話す（「自分の場合は〜」「僕を例に挙げると〜」）

---

## 3. 文体・トーン

- **カジュアルな話し言葉**。友人に話すような自然な口語体
- 断定しすぎない。**「〜かなと」「〜かなって」「〜ですかね」** で柔らかく締める
- 気づきや発見は **「〜なんですよね」** で共感を引き出す形にする
- 面白いと思ったことは **「〜が好き」「〜が楽しい」** と素直に表現する
- 自己開示をためらわない。コミュ障・失敗・弱点を正直に書いてOK

### よく使う語尾・フレーズ
| フレーズ | 用途 |
|---|---|
| `〜なんですよね` | 共感・気づきの共有 |
| `〜かなと` / `〜かなって` | 柔らかい結論・考えの提示 |
| `〜ですかね` | 軽い断定・問いかけ |
| `〜という話で` | 前の話のまとめ・接続 |
| `〜というわけで` | 本題への導入 |
| `逆に言えば〜` | 対比・裏返しの視点 |
| `むしろ〜` | 実は逆という強調 |
| `まあ〜` | 軽くフォロー・ひと呼吸 |
| `〜みたいな` | 例え・ゆるい比喩 |
| `〜からこそ` | 理由の強調 |

---

## 4. 文の長さとリズム

- **1文は短く**。長い文は途中で区切って改行する
- **キーフレーズは1行に独立させる**（前後に空行を入れる）
  ```
  距離は、何を主役として立ち上げるかを決める選択だった。
  ```
- **読点（、）の代わりに改行**で呼吸を入れることが多い
- 箇条書きは「・」か「→」を使う。番号付きリストは①②③か1.2.3.

---

## 5. 気づきの伝え方

- **気づきは後から来る**パターンで書く
  - 「撮り終わってから気づいた」
  - 「帰ってきて写真を並べて初めて〜って気づいた」
  - 「並べてみたらそうなってた、という話で」
- 「気合いを入れてやった結果こうなった」より「なんとなくやってたらこうだった」の方がリアル
- 抽象概念を説明するときは **必ず自分の具体例を添える**

---

## 6. 記事の構成

### 基本パターン
```
① 書き出し（「どうも」+ 近況・文脈）
② 本題の提示（今日のテーマ・気になったこと）
③ 本文（セクション見出し付きで複数）
④ まとめ / おわりに
⑤ 締め（一部記事のみ）
```

### セクション見出し
- 見出しは **体言止め** または **短い問いかけ** 形式
  - ○ `撮り終わってから気づいたこと`
  - ○ `抜けを作りやすいジャンル・時間帯`
  - ○ `個性は"作る"というより、にじみ出るもの`
- 見出しの下にすぐ本文を続ける（冗長な前置き不要）

### まとめ
- 「まとめ」か「おわりに」で締める
- **一言でズバッと結論を言う**スタイル
  - 例: `感想を一言でまとめると、最高でした。それに尽きる。`
- まとめが長くなる場合は、核心だけ残して削る

### 締めの挨拶（スナップ継続記事など定期記事）
- `では、今日はここまで。読んで頂きありがとうございました。`

---

## 7. 強調・演出テクニック

- 強調したいフレーズは **「 」（かぎかっこ）** で囲む
- 引用・名言は独立した行に置いて前後に空行
  ```
  「ひとまず、量のない質はない」
  ```
- 笑いや照れは **（笑）** または **（〜）** で括弧書きに
- 読者への問いかけ → 自分で答えるスタイルを使う
  ```
  「抜けって何？」となりました。
  何となく雰囲気では分かるけど、ちゃんと説明しろと言われると意外と難しい。
  というわけで、自分なりに整理してみました。
  ```

---

## 8. 内容・テーマの扱い方

- **写真・カメラ**が主軸テーマ（スナップ、ポートレート、機材レビュー）
- 技術論だけで終わらず、**自分の感覚・哲学につなげる**
- スポーツ・音楽・カルチャーとの比較を使う（横乗り、筋トレ、音楽など）
- 「上手くなる方法」より **「自分がどう感じたか・何に気づいたか」** を軸にする
- 弱点・失敗を隠さず、**そこから何を学んだか**に繋げる

---

## 9. 避けるべき表現・NG事項

| NG | 理由 |
|---|---|
| `〜でございます`（過剰な丁寧語） | 文体が浮く（ユーモアで使う場合は例外） |
| `〜と思われます` / `〜と考えられます` | 評論文っぽくて距離感が出る |
| `非常に` / `とても` の多用 | 口語なら「めちゃくちゃ」「かなり」の方が合う |
| 長い一文に情報を詰め込む | リズムが壊れる。句読点より改行を優先 |
| 根拠のない断定 | 「〜かなと」で柔らかくする |
| 本文に関係のない前置きが長い | 「どうも」の後はすぐ本題へ |
| 「〜ということで、今回は〜について書いていきたいと思います」 | 典型的な冗長パターン。使わない |
| 「いかがでしたか？」「参考になれば幸いです」 | ブログ定型文。使わない |
| `〜やん` などの関西弁的語尾 | 一人称・文体に合わない。`〜たよな` `〜てたな` に統一 |
| 文頭の `で、` | 軽すぎる印象を与える接続詞。削るか別の接続に変える |
| `〜と思ってて、` で箇条書きや本文につなげる | 言い切れていない。`〜。` で一度切る |

---

## 10. 参考：文体の雰囲気を一言で

> 「撮り終わってから気づいた」を地でいく、後から言語化するスタイル。
> 理屈を語るより感覚を共有する。結論より過程。完成より気づき。

## 11. 作業ルール：フィードバックの反映

- **構成や文体についてフィードバックをもらったら、抽象化した学びをこのCLAUDE.mdに反映する**
- 具体的な修正（この一文を直して）だけで終わらず、「なぜその表現がNGか」を汎用ルールとして9節のNGテーブルや各節に追記する
- フィードバックの抽象化の手順：
  1. 指摘された箇所を修正する
  2. 「この指摘の背後にある原則は何か」を考える
  3. 同じミスを防ぐルールとしてCLAUDE.mdに追記する
